# Hybrid: WENO solver (data) + Causal PINN + Transformer (model)

Fixes the sketch code:
- Data is generated by a **PDE-consistent** solver: 1D **viscous Burgers + reaction** with **WENO5** for convection.
- Transformer sees a real **spatial sequence**: Nx points at a fixed (case,time).
- Causal training uses **time-window gating** (later-time physics losses are downweighted until earlier windows converge).
- PINN loss includes **physics residual** (Burgers+reaction) computed by autograd.

## Install
pip install numpy pandas torch matplotlib

## 1) Generate dataset
python sim/build_dataset.py

## 2) Train
python train.py

## 3) Evaluate
python eval.py


## Regime labeling
We add proxy regime labels from solver outputs:
- peak_u < u_nodet_threshold => no_detonation
- peak_u >= u_det_threshold => detonation_like
- else => deflagration_like
Run-up time is first time max(u(t)) exceeds u_runup_threshold.


## Upgrades: solver-mismatch OOD + ablation
This upgraded version adds:
1) **OOD solver mismatch** split: test data generated with different (nu,k,E) than training.
2) **Ablation modes**:
   - data-only (no physics, no causal/TV)
   - data + physics (no causal/TV)
   - data + causal/TV (no physics)
   - full (data + physics + causal/TV)

Use:
  python sim/build_dataset.py
  python train.py --mode full
  python train.py --mode data_only
  python train.py --mode no_phys
  python train.py --mode no_causal
  python eval.py --mode full



## Conv1D baseline
Conv1D baseline = 1D convolution neural network along the spatial grid x.
It mixes information from nearby points (local receptive field), often strong for 1D PDEs.
